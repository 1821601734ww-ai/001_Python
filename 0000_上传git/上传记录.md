## 第 1 步：在 PyCharm 开启 Git 管理

1. 打开你的项目。
2. 点击中间的 **版本控制**（或菜单 `VCS` / `Git` 相关入口）。
3. 选择 **Git** 作为版本控制工具。

完成后检查：

- 项目文件颜色会变化（表示被 Git 跟踪）。
- 底部或右上角会出现 Git 相关按钮。

---

## 第 2 步：确认本地仓库初始化成功

在 PyCharm Terminal 运行：

```bash
git status
```

预期：

- 能看到当前分支（通常是 `main` 或 `master`）。
- 能看到文件状态（如 `untracked files`）。

---

## 第 3 步：关联远程 GitHub 仓库

### 方法 A（推荐新手）：PyCharm 图形界面

1. 打开 Git 菜单，选择 `Manage Remotes`（管理远程仓库）。
2. 点击 `+` 添加远程：
   - Name: `origin`
   - URL: 你的 GitHub 仓库地址（例如 `https://github.com/用户名/仓库名.git`）
3. 保存。

### 方法 B：终端命令

```bash
git remote add origin 你的仓库地址
git remote -v
```

预期：

- 能看到 `origin` 对应你的 GitHub 地址。

---

## 第 4 步：第一次提交（commit）

1. 在 PyCharm 里打开 **Commit** 面板。
2. 勾选要提交的文件（通常先全选）。
3. 提交信息可写：`初始化项目`。
4. 点击 `Commit` 或 `Commit and Push`。

如果用终端：

```bash
git add .
git commit -m "初始化项目"
```

---

## 第 5 步：推送到 GitHub（push）

如果是第一次推送，建议在终端执行：

```bash
git push -u origin main
```

如果你的分支是 `master`，改成：

```bash
git push -u origin master
```

说明：

- `-u` 会建立本地分支和远程分支的跟踪关系，后续只要 `git push` 即可。

---

## 第 6 步：验证是否成功

1. 刷新 GitHub 仓库网页。
2. 能看到你本地的文件和最新提交记录。

终端验证：

```bash
git status
```

预期：

- 显示 `working tree clean`（工作区干净）。

---

## 常见报错与处理

### 1) `remote origin already exists`

说明已经加过远程。可先查看：

```bash
git remote -v
```

如需重设：

```bash
git remote set-url origin 你的仓库地址
```

### 2) `src refspec main does not match any`

通常是还没 commit，或分支名不是 `main`。先执行提交，再确认当前分支名。

### 3) 认证失败（Authentication failed）

- HTTPS 方式下，GitHub 通常需要 Token，不是账号密码。
- 可在 PyCharm 重新登录 GitHub 账号后再试。
